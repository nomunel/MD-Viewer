# 要件

## 目的

Markdown Viewer は、ローカルの Markdown ドキュメントを軽量なデスクトップアプリとして閲覧するための Viewer です。

ブラウザの File System Access API やローカルサーバー起動に依存せず、OS 標準のフォルダ選択、安定したローカルファイルアクセス、関連付けアプリでの `.md` オープンを提供します。

## 対象ユーザー

- AI Agent と一緒に Markdown ドキュメントを作成・保守するユーザー
- 複数の Markdown ドキュメントルートを切り替えて読みたいユーザー
- ローカルファイルを外部サービスへアップロードせずに閲覧したいユーザー

## 機能要件

- ユーザーは OS 標準のフォルダ選択 UI でドキュメントルートを登録できる。
- 登録したドキュメントルート配下の `.md` ファイルを再帰的に一覧化できる。
- ルート配下のドット始まりファイル・フォルダは一覧化対象外にできる。
- ドキュメントルート自体は `.docs` や `.project-docs` のようなドット始まりでも登録できる。
- `Preview.html` は Markdown ページとして扱わない。
- 左側のツリーからページを移動できる。
- `README.md` の先頭 `#` 見出しをドキュメントタイトルとして表示できる。
- ヘッダーのサブタイトルにドキュメントルートの絶対Pathとページ数を表示できる。
- ページタイトル、Path、本文を対象に検索できる。
- Markdown 内の相対 `.md` リンクを Viewer 内遷移として扱える。
- Markdown 内の相対画像を読み込んで表示できる。
- Mermaid コードブロックを図として描画できる。
- ライトテーマとダークテーマを切り替えられる。
- Viewer のウィンドウサイズと表示位置を次回起動時に復元できる。
- 現在ページのルートフォルダ名付き相対Pathをコピーできる。
- 現在ページの `.md` ファイルを OS の関連付けアプリで開ける。
- 登録済みドキュメント履歴を保存し、次回起動時に再利用できる。
- 履歴から開いたときに参照できなくなったドキュメントは履歴から削除できる。

## 非機能要件

- 通常利用ではローカルサーバーや常駐コンソールを必要としない。
- 通常利用ではブラウザの読み取り権限切れに影響されない。
- Windows 向け配布は `md-viewer.exe` 単体を基本とし、インストーラーを必要としない。
- UI は日本語利用を前提にする。
- Markdown ファイルは Viewer から編集しない。
- 既存の `viewer.html` 直開き方式は開発・確認用フォールバックとして維持する。
- Electron のようにブラウザエンジン一式を同梱せず、Tauri による軽量配布を目指す。

## 制約

- Tauri 版のビルドには Rust/Cargo が必要。
- Markdown レンダラーは独自実装のため、CommonMark や GitHub Flavored Markdown の完全互換ではない。
- リモート画像はブラウザ/WebView が直接読み込める URL のみ表示する。
