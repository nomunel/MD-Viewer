# Viewer 利用者ガイド

このガイドは、Markdown Viewer でローカルの Markdown ドキュメントを読む人向けの手順です。

## ドキュメントを登録する

1. 右上の `ドキュメント一覧` を開きます。
2. `新規フォルダ選択` を押します。
3. OS 標準のフォルダ選択画面で、Markdown ドキュメントのルートフォルダを選びます。
4. 選択したフォルダ配下の `.md` ファイルが左側のツリーに表示されます。

ドキュメントルートは `.docs` や `.project-docs` のようなドット始まりのフォルダでも選択できます。

ルート配下のファイルやフォルダで、名前が `.` から始まるものは一覧化の対象外です。

## ページを読む

左側のページ一覧から Markdown ファイルを選ぶと、本文が中央に表示されます。

`README.md` がある場合は、先頭の `# 見出し` がドキュメント名として使われます。ヘッダーのサブタイトルには、登録したドキュメントルートの絶対Pathとページ数が表示されます。

Viewer のウィンドウサイズと表示位置は、アプリ終了時に保存され、次回起動時に復元されます。

## 検索する

左上の検索欄にキーワードを入力して検索できます。対象はページタイトル、Path、本文です。

`Ctrl+K` で検索欄にフォーカスできます。

検索結果では一致箇所がハイライト表示されます。検索結果からページを開くと、ページ本文内の一致箇所もハイライトされ、最初の一致箇所まで自動スクロールします。

検索欄の文字を変更または削除すると、本文内のハイライトは解除されます。

## Path をコピーする

パンくずの右端にあるコピーアイコンを押すと、現在ページの相対Pathをコピーできます。

コピーされるPathには、ドキュメントルートフォルダ名も含まれます。

## 関連付けアプリで開く

`開く` ボタンを押すと、現在表示中の `.md` ファイルを OS の関連付けアプリで開きます。

たとえば `.md` が VS Code に関連付けられていれば VS Code で開きます。特定エディタを Viewer 側で指定するのではなく、各利用者の OS 設定に委ねます。

## 履歴から開く

登録済みドキュメントは `ドキュメント一覧` に履歴として残ります。

フォルダ移動やリネームなどで履歴のPathを参照できなくなった場合、その履歴を開こうとしたタイミングで一覧から削除されます。

## ブラウザ版フォールバック

`viewer.html` を Chrome または Edge で直接開くと、従来の File System Access API 方式で動作します。

この方式では、ブラウザ再起動後にフォルダの読み取り権限を再確認されることがあります。通常利用では Tauri アプリ版を使ってください。
