# Viewer 利用ガイド

このガイドは、Markdown Viewer を使ってローカルドキュメントを閲覧する人向けの手順書です。

## 前提

Markdown Viewer は、PowerShell のローカルサーバーを使ってローカルフォルダを読み取ります。

`viewer.html` を直接開いた場合は、File System Access API を使うフォールバック動作になります。

## 起動する

1. `docs-preview.cmd` を起動します。
2. ブラウザで Viewer が開いたら `ドキュメント一覧` を開きます。
3. ドキュメントルートの絶対Pathを入力して `追加` を押します。
4. `参照` を押すと、PowerShell 側のフォルダ選択ダイアログからPathを指定できます。

フォルダを登録すると、Viewer は配下の Markdown ファイルを読み込み、左側にページ一覧を表示します。

## ドキュメントを開く

`ドキュメント一覧` には、過去に開いたフォルダが履歴として表示されます。

履歴内のドキュメントをクリックすると、そのフォルダを再度開きます。読み取り権限が失効している場合は、ブラウザの確認に従って再許可してください。

フォルダ移動やリネームなどで参照できなくなった履歴は、ドキュメント一覧から開こうとしたタイミングで自動的に削除されます。

別のフォルダを追加したい場合は、絶対Pathを入力して `追加` を押すか、`参照` からフォルダを選びます。

履歴から不要なドキュメントを消したい場合は、ドキュメント行の削除ボタンを押します。

## タブ名とヘッダー表示

ブラウザタブ名には、現在開いているドキュメント名が表示されます。

ヘッダーのタイトルには、選択フォルダの `README.md` にある最初の `# 見出し` が使われます。`README.md` がない場合や見出しがない場合は、フォルダ名が使われます。

タイトル横のサブタイトルには、選択フォルダ名と検出ページ数が表示されます。

## 複数タブで使う

この Viewer は、現在選択中のドキュメントとページをタブごとに保存します。

そのため、複数のブラウザタブで同じ `viewer.html` を開き、それぞれ別のドキュメントを表示できます。リロードしても、そのタブで最後に開いていたドキュメントとページに戻ります。

ドキュメント履歴、テーマ、サイドバー幅、ツリー展開状態はブラウザ内で共有されます。

エディタ選択もブラウザ内で共有されます。別タブで `VS Code`、`Cursor`、`Windsurf` の選択を変えると、他のタブにも反映されます。

## ブックマークから開く

URL には現在のドキュメントとページが反映されます。

ブックマークすると、次回アクセス時に同じドキュメントとページから開けます。

ただし、対象ドキュメントがブラウザのドキュメント履歴に残っている必要があります。ブラウザの権限が切れている場合は、再度読み取り許可が求められることがあります。

## ページを移動する

左側のページ一覧から Markdown ファイルを選択します。

ページ一覧はフォルダ構造に沿って表示されます。上部のボタンで全展開、全折りたたみができます。

Markdown 内の `.md` への相対リンクも Viewer 内で開きます。リンクにハッシュが付いている場合は、対応する見出し位置へスクロールします。

## 検索する

左上の検索欄にキーワードを入力して `検索` を押すと、ページタイトル、パス、本文を対象に検索します。

`Ctrl+K` で検索欄にフォーカスできます。

検索欄に文字を入力している間は、左側のページ一覧も絞り込まれます。

## 表示を調整する

- `再読込`: 選択中ドキュメントの Markdown 一覧と現在ページを再読み込みします。
- `テーマ`: ライトテーマとダークテーマを切り替えます。
- パンくず横のコピーアイコン: ルートフォルダ名を含む現在ページの相対パスをクリップボードにコピーします。
- `開く`: 選択した外部エディタで現在ページを開きます。
- 右下の上矢印ボタン: ページ先頭へ戻ります。
- サイドバー境界のドラッグ: ページ一覧の幅を変更します。

## 外部エディタで開く

右上のプルダウンで `VS Code`、`Cursor`、`Windsurf` のいずれかを選び、`開く` を押すと現在ページを外部エディタで開きます。

PowerShell ローカルサーバー方式では、ドキュメント登録時の絶対Pathを使うため、外部エディタ用のPathを追加登録する必要はありません。

`viewer.html` を直接開いたフォールバック動作では、各ドキュメントと各エディタの組み合わせごとに、初回だけドキュメントルートの絶対Path入力が必要です。

絶対Pathを登録したドキュメントは、ヘッダーのサブタイトルやドキュメント一覧のメタ情報でもフォルダ名の代わりに絶対Pathが表示されます。

外部エディタで開く機能は、各エディタの URL scheme が OS に登録されている必要があります。未登録の環境では、ブラウザが開けないか確認ダイアログを表示します。

## 困ったとき

ドキュメントが開けない場合は、ブラウザの読み取り権限が失効している可能性があります。ドキュメント履歴から開き直し、権限確認が出たら許可してください。

Markdown ファイルが見つからない場合は、選択したフォルダの中に `.md` ファイルがあるか確認してください。

画像が表示されない場合は、Markdown ファイルから見た相対パスが正しいか確認してください。

外部エディタで開けない場合は、対象エディタがインストールされているか、URL scheme が登録されているかを確認してください。
