# Viewer 利用ガイド

このガイドは、Markdown Viewer を使ってローカル Project 文書を閲覧する人向けの手順書です。

## 前提

Markdown Viewer は、ローカルフォルダをブラウザから読み取るために File System Access API を使います。

主な対象ブラウザは Chrome または Edge です。Firefox や Safari では、フォルダ選択を使った通常運用ができない場合があります。

## 起動する

1. Chrome または Edge で `MD-Viewer.html` を開きます。
2. 初回表示で `Project一覧` を開きます。
3. `新規フォルダ選択` または画面内の `Markdown フォルダを選択` から対象フォルダを選びます。
4. ブラウザの権限確認が出たら、読み取りを許可します。

フォルダを選ぶと、Viewer は配下の Markdown ファイルを読み込み、左側にページ一覧を表示します。

## Project を開く

`Project一覧` には、過去に開いたフォルダが履歴として表示されます。

履歴内の Project をクリックすると、そのフォルダを再度開きます。読み取り権限が失効している場合は、ブラウザの確認に従って再許可してください。

別のフォルダを追加したい場合は、`新規フォルダ選択` を押します。

履歴から不要な Project を消したい場合は、Project 行の削除ボタンを押します。

## タブ名とヘッダー表示

ブラウザタブ名には、現在開いている Project 名が表示されます。

ヘッダーのタイトルには、選択フォルダの `README.md` にある最初の `# 見出し` が使われます。`README.md` がない場合や見出しがない場合は、フォルダ名が使われます。

タイトル横のサブタイトルには、選択フォルダ名と検出ページ数が表示されます。

## 複数タブで使う

この Viewer は、現在選択中の Project とページをタブごとに保存します。

そのため、複数のブラウザタブで同じ `MD-Viewer.html` を開き、それぞれ別の Project を表示できます。リロードしても、そのタブで最後に開いていた Project とページに戻ります。

Project 履歴、テーマ、サイドバー幅、ツリー展開状態はブラウザ内で共有されます。

エディタ選択もブラウザ内で共有されます。別タブで `VS Code`、`Cursor`、`Windsurf` の選択を変えると、他のタブにも反映されます。

## ブックマークから開く

URL には現在の Project とページが反映されます。

ブックマークすると、次回アクセス時に同じ Project とページから開けます。

ただし、対象 Project がブラウザの Project 履歴に残っている必要があります。ブラウザの権限が切れている場合は、再度読み取り許可が求められることがあります。

## ページを移動する

左側のページ一覧から Markdown ファイルを選択します。

ページ一覧はフォルダ構造に沿って表示されます。上部のボタンで全展開、全折りたたみができます。

Markdown 内の `.md` への相対リンクも Viewer 内で開きます。リンクにハッシュが付いている場合は、対応する見出し位置へスクロールします。

## 検索する

左上の検索欄にキーワードを入力して `検索` を押すと、ページタイトル、パス、本文を対象に検索します。

`Ctrl+K` で検索欄にフォーカスできます。

検索欄に文字を入力している間は、左側のページ一覧も絞り込まれます。

## 表示を調整する

- `再読込`: 選択中 Project の Markdown 一覧と現在ページを再読み込みします。
- `テーマ`: ライトテーマとダークテーマを切り替えます。
- パンくず横のコピーアイコン: ルートフォルダ名を含む現在ページの相対パスをクリップボードにコピーします。
- `開く`: 選択した外部エディタで現在ページを開きます。
- 右下の上矢印ボタン: ページ先頭へ戻ります。
- サイドバー境界のドラッグ: ページ一覧の幅を変更します。

## 外部エディタで開く

右上のプルダウンで `VS Code`、`Cursor`、`Windsurf` のいずれかを選び、`開く` を押すと現在ページを外部エディタで開きます。

各 Project と各エディタの組み合わせごとに、初回だけドキュメントルートの絶対Path入力が必要です。入力したPathは Project 履歴に保存され、次回以降は同じPathを使います。

入力するPathは、選択中のドキュメントルートを指す絶対Pathです。現在の Markdown ファイルまで含むPathを入力した場合でも、Viewer は現在ページの相対パスと照合してルートPathへ補正します。

絶対Pathを登録した Project は、ヘッダーのサブタイトルや Project一覧のメタ情報でもフォルダ名の代わりに絶対Pathが表示されます。

外部エディタで開く機能は、各エディタの URL scheme が OS に登録されている必要があります。未登録の環境では、ブラウザが開けないか確認ダイアログを表示します。

## 困ったとき

Project が開けない場合は、ブラウザの読み取り権限が失効している可能性があります。Project 履歴から開き直し、権限確認が出たら許可してください。

Markdown ファイルが見つからない場合は、選択したフォルダの中に `.md` ファイルがあるか確認してください。

画像が表示されない場合は、Markdown ファイルから見た相対パスが正しいか確認してください。

外部エディタで開けない場合は、対象エディタがインストールされているか、URL scheme が登録されているかを確認してください。
