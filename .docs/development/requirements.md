# 要件

## 目的

Markdown Viewer は、ローカル Project 内の Markdown 文書を軽量に閲覧するためのブラウザアプリとして提供する。

ビルドやサーバー起動を必須にせず、`MD-Viewer.html` を直接開いて使えることを重視する。

## 対象ユーザー

- Project 文書を Markdown で管理している開発者
- AI との作業で生成された Markdown ドキュメントをまとめて閲覧したいユーザー
- ローカルファイルを外部サービスにアップロードせず確認したいユーザー

## 機能要件

- ユーザーはローカルフォルダを選択できる。
- 選択フォルダ内の `.md` ファイルを再帰的に検出できる。
- 選択ルート配下のドット始まりのファイルまたはフォルダはインデックス対象から除外する。
- 選択ルートフォルダ自体は、`.docs` や `.project-docs` のようなドット始まりの名前でも利用できる。
- `Preview.html` は Markdown ページとして扱わない。
- 検出した Markdown をフォルダ構造に沿ったツリーで表示できる。
- ユーザーはツリーから Markdown ページを開ける。
- `README.md` の先頭見出しを Project タイトルとして表示できる。
- ヘッダーのサブタイトルには選択フォルダ名とページ数を表示できる。
- Project 履歴を保存し、再利用できる。
- 複数タブで別々の Project を開いても、リロード時に各タブの Project を維持できる。
- URL パラメータから Project とページを復元できる。
- ブラウザタブ名に現在の Project 名を表示できる。
- 現在ページのルートフォルダ名付き相対パスをコピーできる。
- 現在ページを選択した外部エディタで開ける。
- Markdown 内の相対 `.md` リンクを Viewer 内遷移として扱える。
- Markdown 内の相対画像を読み込んで表示できる。
- Mermaid コードブロックを図として描画できる。
- ページタイトル、パス、本文を対象に検索できる。
- ライトテーマとダークテーマを切り替えられる。
- サイドバー幅を変更して保存できる。

## 非機能要件

- 外部サーバーを必要としない。
- 読み取り専用でローカルフォルダを扱う。
- Markdown ファイルは編集しない。
- 外部エディタ連携は URL scheme を使い、Viewer 自体はファイルを書き換えない。
- UI は日本語利用を前提にする。
- Chrome または Edge の File System Access API 対応環境で動作する。
- Project 履歴はブラウザローカルに保存する。

## 制約

- File System Access API 非対応ブラウザでは通常運用できない。
- 任意の絶対Path文字列をフォルダ選択ダイアログの初期フォルダとして指定することはできない。
- Markdown パーサーは独自実装のため、CommonMark や GitHub Flavored Markdown の完全互換ではない。
- リモート画像はブラウザが直接読み込める URL のみ表示する。
- ローカル相対画像は選択フォルダ内のファイルとして解決する。
